# OrderBook Density Scanner

Веб-сканер, который группирует фьючерсные монеты по плотности стакана в диапазоне ±X% от цены.

Работает прямо в браузере — один HTML-файл, без сервера.

Автор: https://t.me/nothinice1

---

## Что делает сканер

Для каждого тикера:

1. Загружает стакан (bids и asks)
2. Вычисляет mid-price:
   (bestBid + bestAsk) / 2
3. Считает объёмы (в USD) в коридоре ±X%:
   - Bids (LONG, зелёная часть):  
     от mid × (1 − X%) до mid
   - Asks (SHORT, красная часть):  
     от mid до mid × (1 + X%)
4. Итоговая ликвидность:
   min(bidLiquidity, askLiquidity)

Это консервативная метрика — она показывает, где **обе стороны стакана плотные**.

---

## Что такое Coverage (покрытие диапазона)

Публичные API бирж не отдают полный стакан.  
Они возвращают только ограниченное число уровней (например, Binance ≈ 1000, Bybit ≈ 200).

Поэтому сканер показывает:

CoverageDown / CoverageUp

Это означает:
> какую часть диапазона ±X% реально покрывают доступные данные.

Примеры:

| Coverage | Значение |
|--------|--------|
| 2.00% / 2.00% | Диапазон покрыт полностью |
| 0.35% / 0.40% ⚠️ | Видна только часть диапазона |

На BTC и ETH шаг цены очень маленький, поэтому даже 1000 уровней могут покрывать всего 0.1–0.4% — это нормальное ограничение API.

## Ошибки 429 (Too Many Requests)

Если появляется:
HTTP 429 – Too many requests  
это значит, что биржа заблокировала ваш IP из-за слишком частых запросов.

Рекомендуемые настройки:
- Параллельных запросов: 3–5
- Задержка между запросами: 80–200 мс

Если несколько человек используют один сервер — ставьте ещё меньше.

## Важно

Этот инструмент показывает структуру ликвидности стакана.  
Он не предсказывает цену и не является финансовой рекомендацией.

Используйте его для анализа:
- защищённых уровней
- зон манипуляций
- реальной глубины рынка
